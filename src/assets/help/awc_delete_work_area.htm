Remove Work Area

 Top  Previous  Next

You can have as many or as few work areas as you need to manage your work in progress. When you remove work areas, only the Adept reference to the directory is removed from the Adept environment. The underlying documents and directory remain on the disk, untouched by the remove work area command.

 

If there are documents in the work area that were checked out from a library, you must check the documents in before you can delete the work area. If there are missing documents in the work area, that is, documents that were checked out to the work area and then removed from the folder, you will need to repair the missing document by either Release Check Out or by replacing the document from a backup.
