Transmittal: Import/Export

 Top 

 

For easier management of companies and contact, the Add/Edit Company dialog offers options to import and export the Transmittal Company and Contact database contents to an Excel spreadsheet (.xlsx).

 

Import

This button will display the Import dialog. Select the Excel spreadsheet to be used for importing. Once a spreadsheet is selected, you must select the worksheet name and the number of header rows in the Excel file. Regardless of the number of header rows, the last row (highest number) will be used for mapping in subsequent dialogs.

 

Once the Scan button is enabled, select the button to scan the spreadsheet and populate the grid. The column Excel Column contains the information specified by the last row of the header rows. The Transmittal Field column is used to map an Excel Column to a Transmittal Field column. The Import Action column is used to specify how the if and how the data will be imported.

 

Note: The Company Name field (S_COMPNAME) is required as no data can exist in the Transmittal database without an association to a company. If the Excel Column has the field value of "S_COMPNAME" it will be automatically mapped to the company name field in the Transmittal database.

 

For most fields, the Import Action is one of the following:

 

Fill (Overwrite) = database field contents will be overwritten with spreadsheet contents. If database field contents are empty, spreadsheet contents are imported to the field.

 

Fill (If empty) = database field contents will be filled only if empty. Existing database field contents will not be changed.

 

Limited to S_COMPNAME field (company name)

Search Key and Fill (If empty) = Database field is used as a control for the importing of this and all other fields from the spreadsheet. Unless this field is set, no spreadsheet importation of any fields will occur. This field, when matched with an existing field in the database, makes no changes to this field but allows other changes to be made to the other related  fields as set for the other fields. If no match is found, the spreadsheet contents is imported, creating a new company in the database and importing related field contents as set in the other fields.

 

Limited to S_CONTNAME field (contact name)

Search Key and Fill (If empty) = Database field is used as a control for the importing of this and other fields from the spreadsheet when coupled with company name field. This field, when matched with an existing field in the database along with a match for company name, makes no changes to this field but allows other changes to be made to the other related fields as set in the other fields. If no match is found for this field, all spreadsheet contents are imported, creating a new contact in the database for the matched company and importing related field contents as set in the other fields.
