Document Type

 Top 

Adept uses the Document Type field to determine which application to associate with each document. By using this Document Type mechanism Adept has the ability to open the appropriate application for different documents even when they each have the same document extension. For example, "nozzle.dwg" could be associated with the application Mechanical Desktop while "isoplan.dwg" could be associated with the application AutoCAD.

 

Setting the Document Type

When you navigate to a new work area or if you are in a work area and press the F5 key, Adept will scan the work area, re-extract data from extraction fields and try to set the document type if it has never been set. Adept also assigns document type to the documents in a library during synchronize. If there is only one document type associated with the document's extension, that document type will automatically be set. If there is more than one document type defined for this extension Adept will check to see if one of them is the default, and if so that one will be assigned. If there is more than one possible document type for a given extension and none of them have been defined as the default, then Adept will pick one of them. The document type of any document can be manually changed by the owner of the document or any user with Edit Data Card rights. To set the document type select the Set Document Type command on the Documents tab.

 

Adept Document Types are associated with an application by the Administrator when configuring applications in the Open Applications dialog.. While the Document Type can be named with any descriptive string, typically the Administrator chooses the same string for the Document Type as is listed in the Windows Registry.

 

Note: The Document Type "Transmittal" is automatically set for Adept Transmittals, and is a reserved Document Type. Do not re-use the name Transmittal in a custom Document Type.
